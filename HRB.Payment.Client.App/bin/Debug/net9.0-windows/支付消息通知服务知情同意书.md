# 支付消息通知服务知情同意书

**生效日期：2026年01月**
**版本号：V1.0**

---

## 一、服务说明

### 1.1 服务定义

**支付消息通知服务**（以下简称"本服务"）是指商家在使用本收银台软件时，可选择启用的一项增值功能。启用后，系统将在检测到支付事件时，自动向商家指定的接口地址推送经过脱敏处理的支付事件数据。

### 1.2 支付事件类型

本服务将推送以下三类支付事件：

1. **用户正在支付**：用户扫码后，支付流程已启动但尚未完成
2. **用户取消支付**：用户主动取消了支付操作
3. **用户支付成功**：支付已成功完成并确认

---

## 二、数据推送说明

### 2.1 推送方式

- **协议**：HTTP/HTTPS POST 请求
- **数据格式**：JSON
- **目标地址**：由商家在设置页面中自行配置的接口URL

### 2.2 推送数据内容

每次推送将包含以下信息：

| 字段名称 | 说明 | 是否脱敏 |
|---------|------|---------|
| 事件类型 | 支付开始/取消/成功 | 否 |
| 支付渠道 | 支付宝/微信 | 否 |
| 支付金额 | 交易金额 | 否 |
| 订单号 | 交易订单标识 | **是（部分脱敏）** |
| 用户昵称 | 付款用户昵称 | **是（部分脱敏）** |
| 用户ID | 付款用户标识 | **是（完全脱敏）** |
| 支付时间 | 事件发生时间 | 否 |

### 2.3 数据脱敏规则

为保护用户隐私，系统将对敏感信息进行脱敏处理：

#### 订单号脱敏
- **脱敏方式**：使用MD5哈希值完全替代真实订单号
- **原始数据**：`20260120123456789012`
- **脱敏后**：`5d41402abc4b2a76b9719d911017c592`（32位MD5哈希值）

#### 用户昵称脱敏
- **中文昵称**：保留首尾各1个字符，中间用星号替代
  - 示例：`张三丰` → `张*丰`
  - 示例：`李四` → `李*`
- **英文/数字昵称**：保留前2位和后2位，中间用星号替代
  - 示例：`JohnSmith` → `Jo****th`

#### 用户ID脱敏
- **脱敏方式**：使用MD5哈希值完全替代真实ID
- **原始数据**：`user123456`
- **脱敏后**：`e10adc3949ba59abbe56e057f20f883e`（32位MD5哈希值）

1. **合法使用**：仅将接收到的数据用于合法的业务目的（如订单管理、数据统计等）
2. **安全存储**：妥善保管接收到的数据，采取必要的安全措施防止泄露

4. **遵守法律**：遵守《个人信息保护法》《数据安全法》等相关法律法规

### 3.2 第三方接收

商家配置的接口地址可能指向：

**重要提示**：商家应确保第三方服务提供商具备合法的数据处理资质，并与其签订数据保护协议。

## 四、技术要求

### 4.1 接口要求

商家配置的接口应满足以下要求：

- 支持HTTP POST请求
- 能够接收JSON格式数据
- 建议使用HTTPS协议以确保传输安全

### 4.2 推送机制

- **实时推送**：事件发生后立即推送，无延迟
- **不保证送达**：采用"发送即忘"模式，系统不等待响应，不进行重试，不保证消息送达
- **无超时限制**：系统不设置超时时间，发送后立即返回

**重要提示**：由于采用"发送即忘"模式，商家应自行确保接口的可用性和稳定性。如需确保消息送达，建议在接口端实现日志记录和监控机制。

## 五、风险提示

### 5.1 数据安全风险

尽管系统已对敏感数据进行脱敏处理，但仍存在以下风险：

1. **网络传输风险**：数据在网络传输过程中可能被截获（建议使用HTTPS）
2. **接口安全风险**：如接口未设置鉴权，可能被恶意访问
3. **第三方风险**：如使用第三方服务，需承担第三方数据泄露风险

### 5.2 服务可用性

本服务采用"发送即忘"模式，可能因以下原因导致消息未送达：

- 网络故障或不稳定
- 目标接口无响应或响应缓慢
- 目标接口返回错误
- 系统维护升级

**免责声明**：因上述原因或任何其他原因导致的消息推送失败、延迟或丢失，本软件不承担责任。商家应自行实现接口端的日志记录和监控机制，以确保业务连续性。
